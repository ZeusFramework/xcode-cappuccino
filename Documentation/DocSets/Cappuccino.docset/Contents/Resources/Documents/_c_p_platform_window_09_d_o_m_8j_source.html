00025  * THIS DOCUMENTATION STOLEN DIRECTLY FROM GOOGLE CLOSURE (licensed under Apache 2)
00026  * 
00027  * Different web browsers have very different keyboard event handling. Most
00028  * importantly is that only certain browsers repeat keydown events:
00029  * IE, Opera, FF/Win32, and Safari 3 repeat keydown events.
00030  * FF/Mac and Safari 2 do not.
00031  *
00032  * For the purposes of this code, "Safari 3" means WebKit 525+, when WebKit
00033  * decided that they should try to match IE's key handling behavior.
00034  * Safari 3.0.4, which shipped with Leopard (WebKit 523), has the
00035  * Safari 2 behavior.
00036  *
00037  * Firefox, Safari, Opera prevent on keypress
00038  *
00039  * IE prevents on keydown
00040  *
00041  * Firefox does not fire keypress for shift, ctrl, alt
00042  * Firefox does fire keydown for shift, ctrl, alt, meta
00043  * Firefox does not repeat keydown for shift, ctrl, alt, meta
00044  *
00045  * Firefox does not fire keypress for up and down in an input
00046  *
00047  * Opera fires keypress for shift, ctrl, alt, meta
00048  * Opera does not repeat keypress for shift, ctrl, alt, meta
00049  *
00050  * Safari 2 and 3 do not fire keypress for shift, ctrl, alt
00051  * Safari 2 does not fire keydown for shift, ctrl, alt
00052  * Safari 3 *does* fire keydown for shift, ctrl, alt
00053  *
00054  * IE provides the keycode for keyup/down events and the charcode (in the
00055  * keycode field) for keypress.
00056  *
00057  * Mozilla provides the keycode for keyup/down and the charcode for keypress
00058  * unless it's a non text modifying key in which case the keycode is provided.
00059  *
00060  * Safari 3 provides the keycode and charcode for all events.
00061  *
00062  * Opera provides the keycode for keyup/down event and either the charcode or
00063  * the keycode (in the keycode field) for keypress events.
00064  *
00065  * Firefox x11 doesn't fire keydown events if a another key is already held down
00066  * until the first key is released. This can cause a key event to be fired with
00067  * a keyCode for the first key and a charCode for the second key.
00068  *
00069  * Safari 2 in keypress (not supported)
00070  *
00071  *        charCode keyCode which
00072  * ENTER:       13      13    13
00073  * F1:       63236   63236 63236

00075  * ...
00076  * p:          112     112   112
00077  * P:           80      80    80
00078  *
00079  * Firefox, keypress:
00080  *
00081  *        charCode keyCode which
00082  * ENTER:        0      13    13
00083  * F1:           0     112     0
00084  * F8:           0     119     0
00085  * ...
